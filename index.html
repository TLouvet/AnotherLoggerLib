ColorLogger is designed to work out of the box with sensible defaults. However, you can customize almost everything to fit your needs.

Default Configuration

  showLevelPrefix: true,
  timestamp: {
    active: true,
    lang: 'en',
  },
  levels: {
    info: { active: true, prefix: '[INFO]', color: '\x1b[36m', bg: '' },

    warning: { active: true, prefix: '[WARNING]', color: '\x1b[33m', bg: '' },

    critical: { active: true, prefix: '[CRITICAL]', color: '\x1b[35m', bg: '' },
  },

üìñ Logger Functions

The Logger class provides several public methods to handle logging efficiently. Below is a list of all available functions, with explanations and usage examples.

1. config(options: DeepPartial<LoggerConfig>)

Configures the Logger with custom settings. This method allows you to adjust the colors, prefixes, and timestamps for all log levels.

The LoggerConfig object defines the structure for all configurable options in the Logger. Here's what it includes:

LoggerConfig Structure


{
  showLevelPrefix: boolean; // Whether to display level prefixes (e.g., [INFO], [ERROR])
  timestamp: {
    active: boolean;        // Enable or disable timestamps
    lang: string;           // Language for timestamps (e.g., 'en', 'fr', 'es')
  };
  levels: {
    info: {
      active: boolean;      // Enable or disable info logs
      prefix: string;       // Prefix for info logs
      color: string;        // Text color (ANSI, Hex, RGB)
      bg?: string;          // Optional background color
    };
    success: { ... };        // Same structure as "info"
    warning: { ... };        // Same structure as "info"
    error: { ... };          // Same structure as "info"
    critical: { ... };       // Same structure as "info"
  };
}

You can provide partial configurations using DeepPartial, and the Logger will merge your configuration with its defaults.

Examples


// Example: Disabling info logs and customizing the warning color
Logger.config({
  levels: {
    info: { active: false },
    warning: { color: '\x1b[38;2;255;165;0m' }, // Orange for warnings
  },
});

// Example: Customizing timestamps
Logger.config({
  timestamp: {
    active: true,
    lang: 'fr', // Display timestamps in French
  },
});

2. usePalette(palette: string)

Quickly switch between predefined color palettes. Available palettes include common and trueColor. 


Logger.usePalette('trueColor');

3. neutral(message: string)

Logs a neutral message without any colors or formatting, similar to console.log. 


Logger.neutral('This is a neutral log message.');

4. Log Levels

Log levels are predefined functions for structured logging. Each level can be customized in the configuration:

  • info(message: string): For general information.
  • success(message: string): For successful operations.
  • warn(message: string): For warnings.
  • error(message: string): For errors.
  • critical(message: string): For critical alerts.

// Examples
Logger.info('This is an informational message.');
Logger.success('This operation was successful!');
Logger.warn('This is a warning!');
Logger.error('Something went wrong.');
Logger.critical('üö® Critical error!');

üé® Colors in Logger

ColorLogger leverages ANSI escape codes to bring vibrant colors to your console. Depending on your terminal, you can use either common ANSI colors or the extended TrueColor system for more flexibility.

Common ANSI Codes

These colors are supported by virtually all terminals. Here are the available options:

Color Code Example
Black \x1b[30m Black
Red \x1b[31m Red
Green \x1b[32m Green
Yellow \x1b[33m Yellow
Blue \x1b[34m Blue
Magenta \x1b[35m Magenta
Cyan \x1b[36m Cyan
White \x1b[37m White (if you don't see it, it probably works)

TrueColor System

For more precise control, the TrueColor system allows you to define colors with full 24-bit RGB values. Use the format \x1b[38;2;R;G;Bm for text color and \x1b[48;2;R;G;Bm for background color. 


        Logger.config({
          levels: {
            info: { color: '\x1b[38;2;0;128;255m' }, // Soft blue
            success: { color: '\x1b[38;2;0;255;128m' }, // Lime green
          },
        });

Hex and RGB Support

For convenience, you can use colors in either Hex format or RGB arrays. The Logger will automatically convert these into TrueColor codes.

Examples


        // Using Hex
        Logger.config({
          levels: {
            warning: { color: '#FFA500' }, // Orange
          },
        });
        
        // Using RGB
        Logger.config({
          levels: {
            critical: { color: [255, 0, 255] }, // Magenta
          },
        });
